# Skein v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A branching command tree with blessed-output regression diffing — bless outputs, recompile, watch divergences flag red — per `docs/superpowers/specs/2026-07-09-skein-design.md`.

**Architecture:** Pure tree logic (`skein-tree.ts`) + pure line diff, behind a `useSkein` composable that reuses the existing worker `replay()` seam and watches the shared compile result for stale/auto-run; a thin `SkeinPanel.vue` outline tab; IndexedDB persistence behind a `SkeinStore` seam with an in-memory test impl.

**Tech Stack:** Existing stack only (Nuxt 4 / Vue composables / Vitest / happy-dom / ifvms replay worker) + `fake-indexeddb` (devDependency, tests only).

## Global Constraints

- TDD per house style: pure logic in plain `.ts` with sibling `*.test.ts` (node env); composable state suites as `*.nuxt.test.ts` (happy-dom).
- No new runtime dependencies. No Pinia. Auto-import-compatible composables; explicit imports for utils (house pattern).
- Statuses: `unblessed | match | diff | stale | error` exactly as specced.
- Per-game keying: `${profile.stateKey}:${activeStoryKey}` (same scheme as test scripts).
- Blessed/last outputs are `normalizeTurnOutput`-normalized text from `TurnRecord.output`; equality is exact string compare.
- The axe CI gate applies to the new tab (real `tree`/`treeitem` roles, keyboard nav).
- Commit after every green task; `yarn test && yarn typecheck` green before each commit.

### Task 1: Pure tree core

**Files:**
- Create: `app/composables/skein-tree.ts`
- Test: `app/composables/skein-tree.test.ts`

**Interfaces (Produces):**
- `type SkeinStatus = 'unblessed' | 'match' | 'diff' | 'stale' | 'error'`
- `interface SkeinNode { id: string; command: string; parentId: string | null; childIds: string[]; blessedOutput?: string; lastOutput?: string; status: SkeinStatus }`
- `interface SkeinTree { root: string; nodes: Record<string, SkeinNode> }`
- `createTree(): SkeinTree` — synthetic root, `command: ''` (boot banner)
- `addPath(tree, commands: string[]): { tree: SkeinTree; leafId: string }` — immutably walks/creates; same command under same parent dedupes (case/whitespace-trimmed compare); sibling order = creation order
- `pathNodes(tree, nodeId): SkeinNode[]` — root..node inclusive
- `pathCommands(tree, nodeId): string[]` — commands root→node EXCLUDING the root's `''`
- `applyRun(tree, leafId, turns: { command: string; output: string }[]): SkeinTree` — turn 0 → root, turn i → path node i; sets `lastOutput`; recomputes status per node: blessed? (equal → `match`, else `diff`) : `unblessed`; path nodes beyond `turns.length-1` (short run) that are blessed → `error`, unblessed stay `unblessed`
- `blessThread(tree, nodeId): SkeinTree` — every path node with a `lastOutput` gets `blessedOutput = lastOutput`, status `match`
- `unblessSubtree(tree, nodeId): SkeinTree` — clears blessing/status→`unblessed` for node + descendants
- `removeSubtree(tree, nodeId): SkeinTree` — removes node + descendants (root refuses)
- `markAllStale(tree): SkeinTree` — every node with `blessedOutput` → status `stale`
- `blessedLeaves(tree): string[]` — deepest blessed node of each blessed path (a blessed node with no blessed descendants)
- `countNodes(tree): number` (excludes root)

**Steps:**
- [ ] Write failing tests covering: root creation; addPath creates chain; addPath dedupes shared prefix and branches at divergence (two scripts sharing 2 commands → 2+1+1 nodes); trimmed-command dedupe; pathCommands excludes root; applyRun alignment incl. banner on root; match/diff transitions; short-run error marking; blessThread copies lastOutput only where present; markAllStale only blessed; blessedLeaves picks deepest; removeSubtree refuses root; immutability (input tree unchanged).
- [ ] Run: `yarn vitest run app/composables/skein-tree.test.ts` → FAIL (module missing).
- [ ] Implement `skein-tree.ts` (pure, immutable updates, id = `s` + counter/random suffix — no Date.now in module top-level; use a local counter + Math.random? Math.random fine in app code).
- [ ] Run → PASS. `git commit -m "feat(skein): pure tree core — paths, bless, diff statuses"`.
